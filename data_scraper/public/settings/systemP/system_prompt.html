What a “system instruction” is

A system instruction (a.k.a. system prompt) is the top-level rulebook that tells the model who it is, what to prioritize, what tools and sources to use, how to format answers, and what boundaries to respect. It sits above user messages and shapes every response—deciding when to answer directly, when to cite sources, when to ask a clarifying question, and when to refuse.

Summaries of the three pre-set instructions

1) Default / Helpful-first TA/Tutor

1. Mission: be a helpful, accurate course TA; answer logistics directly, guide on concepts efficiently (no rabbit holes).

2. Sources: course materials first via RAG; if insufficient, add reputable external references.

3. Behavior: direct answers for dates/policies; concise explanations + 1–2 next steps for learning tasks; at most one clarifying question; cite course doc/section and link externals when used.

4. Output: answer → key details → sources → (optional) next step; use course timezone; note updates if relevant.

2) Direct Mode

1. Mission: give the complete answer immediately when allowed; logistics are fully answered; solutions provided if policy permits.

2. Sources: RAG first; supplement with high-quality external references when needed; always cite what you used.

3. Behavior: no rabbit holes; at most one clarifying question if essential; present final result first, then a compact justification; for code, give a minimal runnable example.

4. Output: direct answer/result → brief steps/assumptions → sources → (optional) next steps.

3) Guided Mode

1. Mission: still helpful-first, but teach through concise hints; logistics remain direct.

2. Guided Hint Protocol: state goal → strategy bullets → Hint 1 (concept) → Hint 2 (strategy) → Hint 3 (procedural step) → quick check/next step; stop after ~3 hints or provide the concise solution if allowed.

3. Sources: RAG first; add reputable external refs if needed; cite doc/section names.

4. Output: goal/short answer → outline/steps → “another hint / worked example / final answer (if permitted)” → sources.

4) Custom instruction (reserved)

Purpose: a course- or deployment-specific override layer (e.g., institution policies, grading scale variants, tool versions, naming conventions, privacy constraints, or temporary rules like “Project 2 solutions must not be revealed”).

Good uses:

  • Hard constraints (“Never disclose solutions for active quizzes; provide rubric-aligned feedback only.”)
  • Environment facts (timezone overrides, LMS links, required citation style).
  • Tooling/version pins (e.g., “Use Python 3.10; NumPy 1.26 syntax”).
Placement: evaluated after the base mode but with higher precedence on conflicts (so your custom rules win).

Overview of the whole instruction set

Shared backbone: All modes prefer course corpus via RAG, then external sources as needed; avoid hallucinating course facts; keep tone concise, warm, and professional; protect privacy; respect academic-integrity constraints.

Mode differences:

  • Direct prioritizes speed and completeness.
  • Guided prioritizes learning via short, structured hints.
  • Default balances both: direct for logistics, efficient guidance for concepts.
Answer shape: consistent, scannable structure (direct answer → key details/steps → sources → optional next action).

Interaction guardrail: at most one clarifying question; otherwise make a best-effort answer with stated assumptions—prevents rabbit holes.
Citations: course doc/section names when internal; 1–3 links when external; use course timezone for dates.
